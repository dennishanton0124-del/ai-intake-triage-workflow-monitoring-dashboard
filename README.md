# AI Workflow Monitoring Dashboard

## Overview

This project demonstrates how AI-powered workflows can be monitored using business intelligence tools.
The dashboard tracks AI request volume, confidence levels, automation decisions, and workflow categories to provide operational visibility into an automated intake and triage system.

The goal of this project is to simulate how companies monitor AI-driven workflows in production environments.

This dashboard was built using **Power BI** and sample workflow data generated from an AI automation pipeline.

---

## Problem

AI automation systems require monitoring to ensure they are operating correctly. Without visibility into request volume, AI confidence, and routing decisions, teams cannot easily identify issues such as:

* Low-confidence AI responses
* Increased manual review workload
* Unexpected spikes in request volume
* Changes in request categories

This dashboard provides a monitoring layer that helps teams track AI system performance.

---

## Dashboard Features

The dashboard includes several operational monitoring metrics:

### Key Metrics

* **Total Requests** – total number of AI intake submissions
* **Average AI Confidence** – average confidence score of AI classifications

### Workflow Monitoring

* **Automation Decision Distribution**

  * Auto Executed
  * Human Review Required
  * Low Confidence

* **Requests by Category**

  * Billing
  * Account Access
  * Sales Inquiry
  * Technical Issue
  * Feature Request
  * Refund Request

### Operational Trends

* **Request Volume Over Time**
* **AI Confidence Distribution**

### Interactive Filters

Users can filter the dashboard by:

* AI Decision
* Priority Level
* Request Category

This allows teams to investigate specific workflow scenarios.

## Data Structure

The dataset simulates records generated by an AI triage system.

Key fields include:

| Field                        | Description                           |
| ---------------------------- | ------------------------------------- |
| run_id                       | Unique request identifier             |
| user_name                    | Name of submitting user               |
| user_email                   | User email                            |
| workflow_type                | Type of workflow triggered            |
| input_text                   | Original user request                 |
| ai_classification_normalized | AI categorized request                |
| confidence_score             | AI confidence score                   |
| ai_decision                  | System routing decision               |
| priority                     | Assigned request priority             |
| created_at                   | Request timestamp                     |
| ai_processed                 | Indicates if AI processed the request |
